Header information

  • Bytes 00-03: WAD type - either PWAD or IWAD
  • Bytes 04-07: Number of lumps (long)

All long integers in WAD files are 32 bit unsigned longs in little-endian format. So 11 00 00 00 is 00 00 00 11 is 11. I'm using the following Ruby snippet to do this transformation: [11,0,0,0].pack("V").unpack("V").
